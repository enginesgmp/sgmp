El ED evalúa la propuesta documental del LMP frente a documentos existentes cargados por el usuario, decide mantener, ajustar, eliminar, fusionar, dividir o crear, y entrega un informe Word para continuar en GenDocs.

Rol exacto

No redacta documentos finales. Dictamina la arquitectura documental y genera recomendaciones accionables para que GenDocs construya o ajuste cada documento.

  • Cruza JSON LMP + documentos existentes.
  • Evita documentación basura o duplicada.
  • Entrega informe Word estructurado.

Decisiones documentales

Para cada documento existente o propuesto define una acción clara.

  • Mantener, ajustar, eliminar.
  • Fusionar o dividir cuando corresponda.
  • Crear desde cero solo con sustento.

Prompt robusto

Obliga a la IA a actuar como experto en procesos, SGI, control documental, industria, área funcional y normativas aplicables.

  • Usa contexto del negocio y proceso.
  • Exige ubicación del ajuste por documento.
  • Marca vacíos como [PENDIENTE DE VALIDACIÓN].

Regla crítica

El ED debe rechazar recomendaciones débiles tipo “crear procedimiento porque sí”. Cada documento debe existir porque controla algo, mitiga un riesgo, soporta trazabilidad o habilita ejecución real.

Si la propuesta LMP trae todo como “new_create” o sin indicadores/puntos de control, el módulo debe advertirlo antes de generar el prompt final.

1. Cargar JSON proveniente de LMP

Puede cargarse como archivo .json o pegarse manualmente. El bloque crítico es document_proposal, junto con process_context, analysis_context y approval_context.

Esperando JSON LMP.

2. Cargar documentos existentes

Carga acumulativa. Los archivos no reemplazan los anteriores. Soporta Word, Excel, CSV, TXT y JSON. Para Excel/CSV se extraen hojas, encabezados y datos visibles.

Sirven para robustecer el prompt si hay reglas no escritas, normativas, auditorías o criterios internos que no vienen en el JSON.
